并查集

定义

​ 并查集是一种树型的数据结构，常用于处理一些不相交集合的合并及查询问题（即所谓的并、查）。例如，我们可以用并查集来判断一个森林中有几棵树、某个节点是否属于某棵树等。

​ 并查集的主要作用是求连通分支数（如果一个图中所有点都存在可达关系（直接或间接相连），则此图的连通分支数为1；如果此图有两大各自全部可达，则此图的连通分支数为2……）

码

  • 所有结点初始祖先为自己。
  • find（x）， 寻找x的祖先。
  • join（x，y），将x，y并到一堆（令其中一个作为另一个的双亲）。
  • 求一组数据中相异的祖先个数可代表为其连通分支数据。

树状数组

树状数组可解决**1.单点修改，单点查询，2.区间修改，单点查询，3.区间查询，区间修改（复杂，建议用线段树处理）**等线段树可解决的大部分问题。

相较于线段树，树状数组处理起来有着更高的效率（指码字快），二者处理问题的时间复杂度大多数情况下相仿（常数级别差异），但树状数组可拓展性较差，处理问题能力有限。

lowbit(x)运算

在进行树状数组的介绍前，我们需要知道啥是lowbit（x），因为树状数组的各项操作都是建立在这基础之上滴。

所谓lowbit(x)，就是求一个非负整数x在二进制表示下，最低位的1及其后面的0构成的数字，如

lowbit(44)=lowbit((101100)2)=(100)2=4lowbit(44) = lowbit((101100)_2) = (100)_2 = 4 lowbit(44)=lowbit((101100)2​)=(100)2​=4

好的，那么我们怎么求这lowbit(x)呢，其实很简单，一条式子足够了。

lowbit(x)=x&(−x)lowbit(x) = x \& (-x) lowbit(x)=x&(−x)

解释：

44的二进制表示为(101100)2(101100)_2(101100)2​，-44的二进制表示为(010100)2(010100)_2(010100)2​，然后我们把44和-44进行按位与运算得到二进制数(000100)2(000100)_2(000100)2​，也就是十进制的4。

注意： -44的二进制表示是44取反后+1。

树状数组结构分析

下面是一个树状数组的结构，至于其如何构建，稍后再说。

对于一个长度为n的数组a[],我们引入一个数组t[x]表示已x为根得子树中叶子节点值的和。

t[1] = a[1]

t[2] = t[1] + a[2]

t[3] = a[3]

t[4]=t[2]+t[3]+a[4]=t[1]+a[2]+t[3]+a[4]=a[1]+a[2]+a[3]+a[4]

…

单独这样来看，似乎找不到什么规律，但是，结合前边提到过的lowbit(x)运算，我们可以发现：

  • 树状数组中节点t[x]t[x]t[x]的父亲节点为t[x+lowbit(x)]t[x+lowbit(x)]t[x+lowbit(x)]。
  • 相邻子树的根节点对应下标的差值也为lowbit，如t[6] = t[7-lowbit(7)],t[4] = t[6-lowbit(6)]…
  • 如果要求前n个元素的和，sum = t[n] + t[n-lowbit(n)] + t[(n-lowbit(n)) - lowbit(n-lowbit(n))] …
  • 如果要求区间[l,r][l,r][l,r]之间元素的和，就做个差分sum(l,r)=sum(r)−sum(l−1)sum(l,r) = sum(r) - sum(l-1)sum(l,r)=sum(r)−sum(l−1)

利用以上几点性质，关于树状数组的基本操作我们就都可以实现啦。

  • t[i]=a[i]−a[i−1]t[i] = a[i] - a[i-1]t[i]=a[i]−a[i−1]
  • a[k] = \sum_\limits{i=1}^{k}t[i]
